Package Management

Under Construction!

Package managers facilitate the installation, patching and removal of software; their importance is difficult to overstate. Each distribution wishing to be part of a larger ecosystem includes a package manager. Beyond operating systems, package managers often exist for other open-source project. Examples include Python, Ruby, and Rust. Finally, there are a few package managers that attempt to be cross-platform, such as Snap, Flatpak and AppImage

What is a Package?

Package managers deal in compiled programs, rather than source code. From the ArchWiki Pacman page: A package is an archive containing:

  • The compiled files of an application
  • Metadata about the application (application name, version, dependencies, etc.)
  • Installation directives for pacman
  • Extra, optional files, such as scripts

The compiled files that form the core of an application are an obvious component of each package; however, in order to manage packages in an efficient manner, each package must contain metadata describing itself. One method to store this information might be to keep it in the executables themselves— version numbers appended to the executable filename, say. This would not be extensible, nor pleasant to use.
