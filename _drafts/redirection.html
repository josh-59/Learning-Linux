Redirection

One of the features that makes the Unix command-line interface powerful is the ability to direct output away from the terminal. Since the terminal is represented as a file, the operations on it and any other file are identical. Thus, by changing the file before calling the command, we can easily capture its output in a regular file; the command does not need to know:

$ lscpu > cpu_stats

This is called redirection. The shell creates the file cpu_stats if it does not exist, and assigns it to be the output file of lscpu. For reasons we'll dive into later, any error messages produced by lscpu will still be printed to the terminal.

We can also direct the output of one command to be the input of another, forming a pipeline:

$ apropos make | less

The redirection operators > and | are not arguments, and are not passed to their respective commands. In fact, apropos does not even know about less!

Finally, if we want to discard the output of a command or pipeline, we can redirect it to /dev/null.
