Programs

A program is a file containing instructions on how to create a process. In Unix-land, program is synonymous with executable file. Most programs are in binary format, meaning that they have been compiled to run directly on the machine's CPU, but others are not compiled, and run directly from source code.

Unix's first binary executable format

a.out

The first Unix binary executable file format was called a.out, shorthand for assembler output.[5] This was a very simple format, consisting of three distinct data segments that coincided with the layout of the resulting process's memory:

  1. Text — Machine instructions
  2. Data — Initial values for global and static variables
  3. BSS — Uninitialized global and static variable values; assigned arithmetic zero by the kernel upon loading

ELF format

The binary format of our time is ELF. Command-line tools for peeking into such a file include:[21]

  • file — Provide a brief description of any filetype
  • size — List section sizes above
  • readelf — Display detailed information about ELF files
  • objdump — "... mostly useful to programmers who are working on compilation tools, as opposed to programmers who just want their program to compile and work."[19]

Script

A script is a text file that is executable. Scripts must begin with the magic characters #!, then the complete pathname of the desired interpreter, as in

Aside:

To execute file in the current working directory, use

./file

#! /bin/bash

or

#! /usr/bin/python.

This line is often called shebang or hashbang. Note that this feature is implemented by the kernel: The kernel itself is passed the script via exec, and goes about executing the interpreter given, with the file as an argument to it.[5] Also, Bash will attempt to execute the file within a subshell, if no hashbang line exists.[22]
