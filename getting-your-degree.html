Discrete Mathematics

At the end of abstraction, there is always math, it seems. Discrete mathematics is not a difficult subject, on par with a first semester in calculus. It will introduce enough logic to give informal proofs, cover graphs and advanced counting techniques, among other thinGiven the inherently discrete nature of the bit, this subject founds all of computer science. The canonical text for computer scientists is Concrete Mathematics, but I prefer Discrete Mathematics - An Open Introduction

Data Structures and Algorithms

After learning a programming language or two, data structures are the next thing to learn in becoming a developer. some understanding of discrete math is a required. To illustrate, the typical methodology in writing a program is to first select the most appropriate data structure, then build the rest of the program around it.

Tending towards academic, given that modern standard libraries include smart implementations of common data structures.

Open Data Structures is a great book on the subject, the best that I personally have found. Advanced studies delve further into mathematics, usually focusing on analysis of algorithms. The canonical text of algorithms is Introduction to Algorithms.

Operating Systems

A very large topic, it discusses operating systems in the abstract. A good, if lighthearted, development is given by Remzi H. Arpaci-Dusseau and Andrea C. Arpaci-Dusseau in their free Operating Systems: Three Easy Pieces.

## Computer Architecture Ah, computer architecture. I loved studying this subject, but honestly wonder how useful it is. It looks into the abstractions presented by hardware itself, such as signed and unsigned numbers, and instruction set architectures. A good development of this material is highly mathematical. I liked Computer Systems: A Programmer's Perspective.

    In two words: Helper Functions!

    If you do wish to contribute a modification to a project, then you have a few responsibilities:

    • Write code that can be read easily
      • Code is read much more often than it is written
    • Write code that can be improved
      • Keep it simple
    • Write code that is modular
      • Account for future development
    But most of all, do one thing and do it well.
