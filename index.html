Conventions:

Command Line

Here, we will use the standard convention to denote the command-line. Commands are presented on a distinct line, and are prefaced by either $ or #:

$ echo hello world
  • $ denotes ordinary-user permissions
  • # denotes superuser (root) permissions

Expressions framed in square-brackets [] are optional:

$ echo [arg1] [arg2]

Repeatable expressions are denoted with ellipses (...):

$ echo [arg ...]

...and italic text may be replaced as appropriate:

$ command [arg ...]

Manual Pages

We will use the standard convention for specifying man pages: The name of the page is followed by its section number, which is placed in parenthesis. For example, cat(1) refers to the manual page in Section 1, while printf(3) refers to the manual page in Section 3.

Code

On occaision, code snippets will be presented. In this work, they will be restricted to the C Programming Language and formatted so: 

    #include <stdio.h>

    int main() {
        printf("Hello world;");
        return 0;
    }

    // This is a comment.  It is ignored by the compiler.
