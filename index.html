How-to

It's worthwhile, from time to time, to take a step back and reconsider the process of learning Linux itself.

  1. What, exactly, is your question?

    Often, the hardest part is just figuring out what your question is. Try to be specific: In computers, things are very precisely defined.

  2. Be Patient.

    Skill accrues. Answer one question at a time.

  3. Reconsider your sources.

    Does it make sense that this person would write about the subject at hand? What relationship do they have to the project? How authoritative are they?

While many books have been intended to introduce GNU/Linux, few are very good, and fewer still are free. It is a large subject, and moves quickly. The good ones focus on a specific facet. Some of my favorites:

  • Debian Administrator's Handbook— A free book written by Raphaël Hertzog and Roland Mas, two Debian developers. It was originally written in French, and has been translated into four other languages, with twenty(!) more translations in-progress.
  • The Linux Programming Interface— The definitive text of the programming interface of the Linux kernel. Michael Kerrisk is the maintainer of the Linux Man Page project (below), and his work is simply terrific. If you want all the gory details of the system-call interface offered by the kernel, as well as sound explanations of its internals, this is your book.
  • Linux Kernel in a Nutshell— A free book by Greg Kroah-Hartman. He says, "I want this book to help bring more people into the Linux kernel development fold. The act of building a customized kernel for your machine is one of the basic tasks needed to become a Linux kernel developer. The more people that try this out, and realize that there is not any real magic behind the whole Linux kernel process, the more people will be willing to jump in and help out in making the kernel the best that it can be."[2]

Man Pages

The most useful asset in the Linux documentation pool is the manual ("man") pages. Man pages provide a concise overview of commands and features, with further documentation existing elsewhere. Learn to use them early, for they are a way of life.

Man pages are accessed at the command line:

$ man cat

This launches the man paging utility, which lets you view and search a page through a read-only interface. Within the man utility, you can search the content by typing a single forward slash, then your search term:

/search-term

Aside:

Incidentally, these controls have become standard for paging utilites.

Then, enter. Jump to the next result by typing a single n. To quit, type q.

The controls of the man paging utility are simple enough: To move down one line, type j; to move up one line, use k. To move down a screenful, press the spacebar.

Conventions:

Links

In this work, links are restricted to those that are both useful and non-obvious.

  • Inline links point out immediately relevant sources
  • Superscript[1] references denote authoritative resources which aren't necessarily of interest

Wikipedia will be considered authoritative (and often, its documentation proves quite good)

Man Pages

Here we will use the standard convention for specifying man pages: The name of the page is followed by its section number, which is placed in parenthesis. For example, cat(1) refers to the manual page in Section 1, while printf(3) refers to the manual page in Section 3.

Command Line

Again, we will use the standard convention to denote the command-line. Commands are presented on a distinct line, and are prefaced by either $ or #:

$ echo hello world
  • $ denotes ordinary-user permissions
  • # denotes superuser permissions

Expressions framed in square-brackets ([]) are optional:

$ echo [arg1] [arg2]

Repeatable expressions are denoted with ellipses (...):

$ echo [arg ...]

Italic text may be replaced as appropriate:

$ command [arg ...]

Code

On occaision, code snippets will be presented. In this work, they will be restricted to the C Programming Language and formatted so: 

    #include <stdio.h>

    int main() {
        printf("Hello world;");
        return 0;
    }

    // This is a comment.  It is ignored by the compiler.
