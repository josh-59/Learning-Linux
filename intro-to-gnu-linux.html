An operating system is a collection of software designed to facilitate the use of a computer. To this end, its two main responsibilities are to support running programs, known as processes, and to maintain persistent data, usually in the form of files. In addition, operating systems ease the task of creating useful programs (i.e., computer programming). Modern GNU/Linux offers the single best programming environment available today.

Unix Architecture

GNU/Linux belongs to a large class of operating systems called Unix-like, sometimes referred to by *nix. The Unix operating system had a clear and well-documented architecture, which has been replicated many times in both open-source and proprietary contexts. The canonical depiction of its architecture is as follows:

Hardware: At the heart of any machine is its hardware. Each piece of hardware has a software interface defined by the engineers that created it. These interfaces are varied at best.

Drivers: Directly interacting with hardware and providing a more-uniform interface to each piece is the task of the system's drivers. This software layer represents the deepest and most remote element of any operating system. While drivers can be installed on an as-needed basis, in Linux the tendency is to include many drivers, and use only those that are required.

Kernel: While the heart of any machine is its hardware, the heart of any Unix-like operating system is its kernel. The kernel uses drivers to direct the actions of the machine, at the behest of processes. Because the expectations that we have for the system are great (running multiple programs concurrently, never losing files nor parts of files, weeks-long stability), this is a large and involved piece of software.

System Calls: In order to permit interaction with the kernel, each Unix-like operating system employs a well-defined set of system calls. Interaction with the kernel is restricted to the system-call interface. From a source-code point of view, these "look" and "act" like ordinary function calls, but their actions are not restricted to user-space; rather, they transcend the software hierarchy, often making changes at the hardware level.

Processes: From The Linux Programming Interface by Michael Kerrisk, "A process is an instance of an executing program."[1] On a *nix system, processes are the only means that a user has to interact with the system. Processes are a critical component of a functioning Unix-like operating system.

The Linux Kernel

Properly speaking, the name Linux refers to the Linux kernel. The Linux kernel was began by Linus Torvalds in 1991, then a graduate student of the University of Helsinki.[1] Following the architecture of the Unix operating system, he wrote an open-source kernel as "...a hobby, won't be big and professional like [the GNU kernel]." He soon released it publicly under the free-and-open-source GNU General Public License, version 2.
