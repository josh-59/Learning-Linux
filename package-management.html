Package Management

Under Construction!

Packages provide a uniform way to install, patch and remove software, and exist in multiple contexts within the FOSS domain. Each distribution wishing to be part of a larger ecosystem includes a package manager. Large programming language ecosystems often offer a package management tool to ease its own use. Examples include Rust's cargo, Python's pip and Ruby's gem.

What is a Package?

A package is a compressed bundle of data that contains everything required to implement a set of related commands or features.[1] In addition, packages contain metadata in the form of files which describe the package; these are crucial to the organization of packages and the automation of their installation.

Let's have a look at Debian's locate. We download the file from a Debian repository mirror, and receive a single file, locate_4.6.0+git+20190209-2_amd64.deb. Note the .deb extension.

This file can be uncompressed in a few ways.

  1. On Mint, by right-clicking and selecting "Extract Here"
  2. From the command line, with traditional tools:
    $ ar x locate_4.6.0+git+20190209-2_amd64.deb

In either case, we find three files, control.tar.xz, data.tar.xz and debian-binary. The debian-binary file lists the package-format version number.[1] Both .tar.xz files are technically uncompressed, but bundled (archived). Let's first look at the data file.

  1. On Mint, right-click and select "Extract Here"
  2. From the command line:
    $ tar -xJf data.tar.xz

Next, let's look at the control file. This archive contains the metadata files; each is plaintext, and can be viewed with cat or less.

$ cat control
Package: locate
Source: findutils
Version: 4.6.0+git+20190209-2
Architecture: amd64
Maintainer: Andreas Metzler
Installed-Size: 421
Depends: findutils (>> 4.2.31-1), libc6 (>= 2.14)
Breaks: slocate (<= 3.1-1.1)
Replaces: findutils (<< 4.2.31-2)
Section: utils
Priority: optional
Multi-Arch: foreign
Homepage: https://savannah.gnu.org/projects/findutils/
Description: maintain and query an index of a directory tree
 updatedb generates an index of files and directories.
 GNU locate can be used to quickly query this index.

This might look familiar: It is the result of querying apt about a particular package:
$ apt show locate returns similar results.

From the ArchWiki pacman page: A package is an archive containing:

  • The compiled files of an application
  • Metadata about the application (application name, version, dependencies, etc.)
  • Installation directives for pacman
  • Extra, optional files, such as scripts
