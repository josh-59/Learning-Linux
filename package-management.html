Package Management

A package is a bundle of data that contains the things required to implement a program, a set of related programs, or a set of related features.[1] It contains the primary binary executable, other assets required by it, a description of the software included, and a list of other dependancies (if any). Packages provide a way to automate the installation of software,[1] and exist in multiple contexts within the FOSS domain.

Advantages

On other operating systems, it is common to download an executable file, then execute it, to install new software. Centralized software management offers a few advantages over this design:

  1. Eliminating Repeated Code

    Packages eliminate the need to include an installation solution, since they abdicate this to a utility (here, apt). In software development, repition is not only avoided, it is also discouraged.

  2. More Controlled Installation

    apt controls where each file is placed. Thereby, a uniformity across the system may be maintained.

  3. Centralized Updating

    Typically, all of the software on a system is managed by the package manager. Thereby, all of the software on a system (that is, the system itself) may be kept up-to-date by a single utility.

Repositories

Collections of packages are called repositories, or sometimes, archives. A repository may be remote (the usual case), or it may be local (e.g., on a thumb-drive).

A repository belonging to a large project, such as Debian's, is often mirrored by other organizations.
