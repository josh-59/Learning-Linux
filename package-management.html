Package Management

Under Construction!

Packages provide a uniform way to distribute, install and patch software, and exist in multiple contexts within the FOSS domain. This essay will cover the big ideas of package management, and explain how they can be used.

What is a Package?

A package is a bundle of data that contains everything required to implement a set of related commands or features. It is a container for software that also contains some metadata, in the form of files, which describes the software it contains.

Let's have a look at Debian's locate, as an example. We download the file from a Debian repository mirror, and receive a single file, locate_4.6.0+git+20190209-2_amd64.deb. Note the .deb extension.

This is a compressed file, and may be uncompressed in a few ways.

  1. On Mint, by right-clicking and selecting "Extract Here"
  2. From the command line, with traditional tools:
    $ ar x locate_4.6.0+git+20190209-2_amd64.deb

In either case, we find two archives, control.tar.xz, and data.tar.xz. Let's first look at the data file.

  1. On Mint, right-click and select "Extract Here"
  2. From the command line:
    $ tar -xJf data.tar.xz

Next, let's look at the control file. This archive contains the metadata files; each is plaintext, and can be viewed with cat or less.

  • Installation directives for pacman
  • Extra, optional files, such as scripts

Package Managers

A package manager is a command-line tool which installs new software, updates existing, and removes unwanted software.

APT

Debain's Advanced Package Tool (APT) is the distributions primary package-management front-end.[3] We can search with,

apt search <query>

dpkg

https://unix.stackexchange.com/questions/138188/easily-unpack-deb-edit-postinst-and-repack-deb

Front-end's

dpkg is not intended as a user interface "front-end." Instead, it is the backend tool of developers and software engineers: It is stable enough to allow them to build on top of it. There are a few pieces of software which are intended as the user's front-end.

  • APT— "Advanced Package Tool (or APT), the main command-line package manager for Debian and its derivatives. It provides command-line tools for searching, managing and querying information about packages, as well as low-level access to all features provided by the libapt-pkg and libapt-inst libraries which higher-level package managers can depend upon."[3]
    • apt install --install-recommends --install-suggests
  • aptitude— "Aptitude is an Ncurses and command-line based front-end to numerous Apt libraries, which are also used by Apt, the default Debian package manager. Aptitude is text based and run from a terminal."[4]
  • synaptic

In addition, distributions often create their own graphical front-end.

  • Mint Install— Written in Python, Mint Install offers software from both Debian and Flatpak repositories.[4]
  • Ubuntu Software Center— An easy-to-use interface that provides a central point for installing and removing software.[5]
