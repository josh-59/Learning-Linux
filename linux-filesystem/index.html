The Linux Filesystem

Under Construction!

Unix began life as a filesystem; on top of this was created the means of interacting with it, including an exec function, shell, text editor and assembler (Kernighan 33-34). Created by Ken Thompson, this small software suite formed the nexus of the time-sharing operating system that would eventually be called Unix.

This file-first approach to operating system design is found in the implementation as well as the history of Unix and its derivatives; concurrently, there is a lot to be said about files. This paper aims to describe the structure of the Unix filesystem, and the various uses that it's put to. We'll start off pretty pedestrian, with regular files and their uses. Then we will progress to directories, which are files that list other files, and eventually cover virtual filesystems, which exist at run-time alone, and are used, for instance, to make available information about running processes.
